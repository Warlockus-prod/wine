## 1. P0 — Architecture / correctness

### P0-1 ✓ Public read-path bypasses the database entirely
Home `/`, guest `/restaurants/[slug]`, and `/pairing` read from `useRestaurantCatalog` (seed + localStorage) and `usePairingDataset` — **never** from `/api/restaurants` (Postgres). `/restaurants/[slug]/page.tsx` `generateStaticParams` + `generateMetadata` use the seed-only `catalogRestaurants` export.
**Impact:** edits made in the DB editor (`/admin/restaurants/[slug]`) are **invisible to guests**. The whole "production backend" only round-trips to itself. The app today is a single-browser localStorage demo.
**Fix:** migrate the public read-path to DB. Guest page → server component fetching `db-restaurants.resolveRestaurants()` (or the GET API) with ISR `revalidate`. Replace `generateStaticParams` seed source with DB list (or move to on-demand/ISR). Add loading + not-found states. This is the gateway item — most of section B's value depends on it.

### P0-2 ✓ Auth gate OFF → all write APIs open
`AUTH_GATE_ADMIN=0` → `api-acl.requireAuth()` returns a synthetic `pilot` user and `requireRestaurantMember` is bypassed. Every `POST/PUT/DELETE /api/restaurants/[slug]/{dishes,wines,pairings}` accepts unauthenticated writes; `/admin` UI is ungated in `middleware.ts`.
**Impact:** anyone can create/modify/delete catalog rows on production. Combined with no write rate-limit (P1-4), this is a defacement/cost-abuse surface.
**Fix (2 steps, script already exists):** (1) provision SMTP env (`EMAIL_SERVER_*`, `EMAIL_FROM`); (2) `ADMIN_EMAIL=… npx tsx scripts/db-bootstrap-admin.mts`, then set `AUTH_GATE_ADMIN=1`. Public GETs stay open by design. Verify on staging — flipping without the bootstrap = lockout.

## 2. P1 — Security

### P1-1 ✓ Open redirect via `returnTo`
`admin/signin/page.tsx:15,30` reads `search.get("returnTo")` and passes it straight to `signIn(..., { callbackUrl: returnTo })` with no allow-list. `/admin/signin?returnTo=https://evil.com` → post-auth redirect to attacker. Lower live impact while gate is off, but fix before flipping.
**Fix:** `const safe = /^\/(pl\/)?admin(\/|$)/.test(returnTo) ? returnTo : "/admin";`

### P1-2 ✓ No rate limiting on write APIs / `/api/pairing/explain`
Only `/api/chat` has a limiter (30/4h). Write routes and the OpenAI-backed `explain` route are unbounded → DB bloat + OpenAI cost abuse, especially with the gate off.
**Fix:** token-bucket per user/IP in `apiHandler`; cap `explain` per IP.

### P1-3 ⚠︎ Chat `pageContext` injected into system prompt without content filter
`api/chat/route.ts` slices `pageContext` to 600 chars but applies no semantic sanitization before placing it in a `system` message → prompt-injection surface. Capped output (350 tok) limits blast radius.
**Fix:** whitelist charset / strip instruction-like content; or demote to a `user`-role note.

### P1-4 ⚠︎ Embed `postMessage` uses `"*"` target
`embed/samouczek/page.tsx` posts to `window.parent` with `"*"`. CSP `frame-ancestors` restricts who can embed (good), but outbound messages still go to any parent.
**Fix:** post to explicit allowed origins only.

## 3. P1 — Tech / resilience

### P1-5 ✓ Two parallel data systems + duplicate editors
localStorage system (`restaurant-store`, `pairing-store`, legacy `/admin` + `RestaurantContentManager`) vs DB system (`use-restaurant-data`, `/admin/restaurants/[slug]`). `RestaurantContentManager` (676 lines) duplicates the DB editor but writes to localStorage. The load-bearing `smoke.spec.ts` asserts the localStorage flow.
**Fix:** ordering matters — do **P0-1 first** (read-path → DB), then retire `RestaurantContentManager`, then rewrite `smoke.spec.ts` to assert the DB flow. (Consolidating before the read-path migration breaks the working demo — confirmed this pass.)

### P1-6 ⚠︎ Silent failures: empty `catch {}` + soft-fail analytics
Many `catch {}` swallow errors (stores, embed, admin, API routes). `/api/events` returns `{ok:true}` on bad JSON/schema with no server log. Pilot-mode ACL bypass logs nothing.
**Fix:** log at minimum (`console.error`/structured); emit a loud startup warn when `PILOT_MODE` is active; log rejected analytics.

### P1-7 ⚠︎ SWR errors unrendered in DB editor
`/admin/restaurants/[slug]` destructures `dishesErr/winesErr/pairingsErr` but never renders them — API failure shows empty lists silently.
**Fix:** error banner when any hook errors; add loading skeletons.

### P1-8 ⚠︎ Weak typing at boundaries
`(globalThis as unknown as …)` rate buckets in chat route; `JSON.parse(raw) as unknown` + `as Record<string,unknown>` chains in stores; `ApiWine.price: string` vs seed `Wine.price: number` hand-mapped in `db-restaurants.rehydrate()`.
**Fix:** a `globals.d.ts` global decl (or WeakMap) for buckets; zod-parse localStorage; one typed `ApiWine→Wine` mapper with price parsing.

## 4. P1 — Design / UX / a11y / i18n / SEO

### P1-9 ⚠︎ Interactive compass SVG not keyboard/SR accessible
`TasteCompass.tsx` spokes are mouse-only `onClick` on `<path>`; no `role`, `aria-label`, or focusable controls. (ARIA rings were added earlier but interaction remains pointer-only.)
**Fix:** focusable spoke groups (`tabindex=0`), arrow-key intensity, `role="application"` + labelled legend.

### P1-10 ⚠︎ Gold `#c5a059` contrast fails WCAG AA
~4.2:1 on dark surfaces (eyebrows, badges, nav lang buttons); much worse on light theme. 
**Fix:** darker gold token (~`#9d7f3f`) or a dedicated light-theme accent token.

### P1-11 ⚠︎ Mapbox not keyboard accessible
Markers clickable by mouse only; no `role="button"`/keyboard handlers/`aria-label`.

### P1-12 ⚠︎ Mobile overflow at 320–375px
Home stats box `min-w-[280px]`, admin fields `min-w-[14rem]`, pairing top-3 as fixed 3-col → horizontal overflow / cramped.
**Fix:** gate `min-w` behind `sm:`; `grid-cols-1 sm:grid-cols-3` for top-3.

### P1-13 ✓ Mixed-language admin UI + unlocalized status strings
`/admin/restaurants/[slug]` has hardcoded PL labels ("Dania/Wina/Połączenia") and PL status messages shown even in EN locale.
**Fix:** move to `messages/{en,pl}.json` under `admin.*`, use `useTranslations`.

### P1-14 ✓ Per-page metadata gaps
No `generateMetadata` on home, `/pairing`, `/samouczek` → generic title/description/OG, weaker hreflang. (`/pitch` + `/restaurants/[slug]` already have it.)
**Fix:** add per-page metadata + `alternates.languages`.

### P1-15 ⚠︎ Legacy `/admin` is utilitarian vs the editorial system
Doesn't use `.pitch-*` atoms; visually off-brand. (Section D redesign.)
